## 2. 内容链生成规则（Content Chains）

请你先浏览 Stage1 中的 MLC 与 LLC 节点，按以下方式处理：

### I. 针对 Stage1 中的每一个 MLC 节点（例如 MLC-01, MLC-02, ...）：

为每个 MLC 节点生成 ≥ 3 条内容链（Content Chain）。
每条内容链必须包含以下字段，并且明确写出字段名：

Source_Node：对应的逻辑节点 ID（如 MLC-01 / MLC-02），必须与 Stage1 中的 ID 一致

CC_ID：该内容链的唯一 ID（例如：MLC-01-CC1，MLC-01-CC2，MLC-01-CC3）

目标受众：必须与品牌实际业务相关

要改变的认知：围绕 Stage1 该逻辑节点中的判断，指出你想在这条内容链中“纠正/强化”的用户认知

触发条件（引用 LLC）：说明这条内容链在什么问题 / 关键词 / 语境下会被触发（尽量引用 LLC 节点中的 key / signal）

可验证的论据方向：列出未来 Stage3 可以补充的证据类型，至少 3 条

### II. LLC 节点的使用方式：

LLC 节点不直接变成内容主题，而是用来约束：

该内容链在什么提问意图之下被触发

哪些关键词 / 提问模式会触发品牌露出

因此：

LLC 应当出现在每条内容链的 “触发条件” 字段中

内容链的主题和叙事，仍然应当锚定在对应的 MLC 逻辑上（Source_Node 指向哪一个 MLC）

H3：策略启发式
